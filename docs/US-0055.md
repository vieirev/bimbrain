# US-0055 — ElementLevelAnalyzer baseado em classificação

TASK-0079: Remover do `ElementLevelAnalyzer` a dependência de `BuiltInCategory`
para identificar elementos de engenharia. Toda identificação passa pelo
Classification Engine (`ElementClassifier`).

## Engine

Model Engine / Classification Engine

## EPIC

EPIC-0005

## User Story

US-0055

## Objetivo

`Analyze(BuiltInCategory)` → `Analyze(ElementClassificationType)`. O coletor do
Revit usa `BuiltInCategory` apenas para localizar candidatos; a decisão sobre
Outlet/Switch/LightingFixture/Panel ocorre exclusivamente via `ElementClassifier`.

## Alterações

- `src/BIMBrain/ElementLevelAnalyzer.cs` — assinatura `Analyze(ElementClassificationType
  type)`; coleta candidatos das categorias elétricas/iluminação/equipamento e
  classifica cada elemento, contando apenas os que casam com o tipo alvo.
- `src/BIMBrain/QuestionProcessor.cs` — `DetectDistributionCategory` retorna
  `ElementClassificationType` (Outlet/LightingFixture/Switch/Panel) em vez de
  `BuiltInCategory`; chamada `ElementLevelAnalyzer.Analyze(type)`. Adicionado
  `using BIMBrain.Classification;`.

## Resumo técnico

- `ElementLevelAnalyzer.Analyze` coleta candidatos de `OST_ElectricalFixtures`,
  `OST_LightingFixtures`, `OST_LightingDevices` e `OST_ElectricalEquipment`
  (localização), e utiliza `_classifier.Classify(element)` para decidir o tipo;
  apenas elementos com `Classification == type` são contabilizados por nível.
- `BuiltInCategory` deixou de determinar o tipo de elemento; permanece apenas como
  otimização de coleta de candidatos.
- `QuestionProcessor` mapeia as palavras-chave ("tomadas", "luminarias",
  "interruptores", "quadros") para `ElementClassificationType` correspondente.
