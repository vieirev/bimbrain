# US-0041 — Reorganização da Ribbon por Engines

## Título

TASK-0065: Reorganização da Ribbon do BIMBrain para refletir a arquitetura
atual do produto e preparar o acesso às próximas funcionalidades (Foundation
Engine, EPIC-0001).

## Objetivo

Reorganizar a Ribbon em painéis por Engine e criar os botões de acesso às
funcionalidades futuras. Nenhuma funcionalidade existente foi removida. Apenas
o botão `Abrir BIMBrain` executa funcionalidade existente; todos os demais
utilizam uma implementação provisória que informa
`"Funcionalidade em desenvolvimento."`.

## Critérios de Aceite

- [x] Ribbon organizada por Engines.
- [x] Nenhum comando existente removido.
- [x] `Abrir BIMBrain` continua funcionando (abre a janela de consultas).
- [x] Demais comandos utilizam `PlaceholderCommand`.
- [x] Arquitetura preparada para evolução futura.

## Detalhes Técnicos

- `PlaceholderCommand` é uma única classe `IExternalCommand` reutilizada por
  todos os botões ainda não implementados (sem duplicação de comandos).
- `App.OnStartup` cria a aba `BIMBrain` e os cinco painéis, instanciando
  `PushButtonData` para cada botão.
- `Abrir BIMBrain` referencia `typeof(Command)` (funcionalidade existente).
- Botões sem ícone próprio utilizam ícones padrão do Revit quando disponíveis
  (`LoadStandardIcon` tenta carregar recursos embutidos de `RevitAPIUI`; ausentes,
  o botão fica apenas com texto).
- `Command.cs` não exigiu alteração (o botão `Abrir BIMBrain` continua apontando
  para a classe existente).
