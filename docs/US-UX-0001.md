# US-UX-0001 — Redesenho comercial da Ribbon

## Título

TASK-UX-0001: Redesenho completo da Ribbon do BIMBrain com aparência de plugin
comercial (UX Refinement, Foundation Engine, EPIC-0001). Não cria nova TASK
numerada; não pertence ao roadmap oficial.

## Objetivo

Redesenhar a Ribbon para melhor organização, espaçamento, menos poluição
visual e uso de ícones, preparando crescimento futuro. Nenhuma funcionalidade
foi alterada e nenhum comando existente foi removido.

## Critérios de Aceite

- [x] Ribbon significativamente mais limpa (4 grupos compactos).
- [x] Botão principal `BIMBrain` em destaque (grande).
- [x] Botões relacionados agrupados e empilhados (`AddStackedItems`).
- [x] Uso de `AddStackedItems` nos grupos Copilot, Engenharia e Ferramentas.
- [x] Todos os botões possuem ícones (`Image` 16×16 e `LargeImage` 32×32).
- [x] Todos os botões possuem `ToolTip`.
- [x] Todos os botões possuem `LongDescription`.
- [x] Nenhuma funcionalidade existente alterada.
- [x] Nenhum comando removido.
- [x] `BIMBrain` (antes "Abrir BIMBrain") continua funcionando.

## Ícones

- Pasta `Resources/Icons16/` (16×16) e `Resources/Icons32/` (32×32).
- Ícones temporários: letra branca sobre cor sólida (genéricos).
- Adicionados ao `.csproj` como `EmbeddedResource` e carregados via
  `Assembly.GetManifestResourceStream("BIMBrain.Resources.<arquivo>")`.
- Toda a Ribbon permanece autocontida em um único DLL (sem arquivos externos).

## Detalhes Técnicos

- `App.OnStartup` cria a aba `BIMBrain` e quatro painéis: `BIMBrain`,
  `Copilot`, `Engenharia`, `Ferramentas`.
- Botão principal: `PushButtonData` único grande com `LargeImage`.
- Demais grupos: `panel.AddStackedItems(pb1, pb2, pb3)` com três
  `PushButtonData` cada.
- `MakePlaceholder(...)` gera `PushButtonData` reutilizável para
  `PlaceholderCommand`, já preenchendo ícone, `ToolTip` e `LongDescription`.
- `LoadIcon(resourceName)` carrega o PNG embutido como `BitmapSource`; retorna
  `null` com segurança se o recurso não for encontrado.
- `PlaceholderCommand` inalterado (exibe `"Funcionalidade em desenvolvimento."`).
