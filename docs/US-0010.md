# US-0010

## Título

Diagnóstico automático do modelo elétrico

## Objetivo

Implementar a primeira análise composta do BIMBrain com capacidade de diagnóstico. O sistema produz um relatório contendo resumo quantitativo do projeto e verificações de completude, resultando em um status geral (OK ou ATENÇÃO). Toda a análise é produzida pela engine do BIMBrain, sem uso de IA.

## Fluxo Principal

1. Usuário pergunta "diagnóstico", "analisar projeto", "analisar modelo", "saúde do projeto" ou "health check".
2. BIMBrain coleta os dados do modelo ativo via métodos existentes (CountByCategory, SumParameter, ModelContext, FilteredElementCollector).
3. Produz relatório com resumo quantitativo (modelos, níveis, tomadas, interruptores, luminárias, quadros, circuitos, ambientes, área, conduítes).
4. Realiza verificações de completude com indicadores ✔ (ok) ou ⚠ (atenção).
5. Exibe status geral: OK ou ATENÇÃO com mensagem explicativa.

## Critério de Aceite

- Nenhuma consulta existente é alterada.
- Diagnóstico reutiliza exclusivamente código existente (CountByCategory, SumParameter, ModelContext).
- Nenhum FilteredElementCollector duplicado — apenas coletores que não possuem método equivalente.
- Nenhuma consulta nova é criada além do diagnóstico.
- O relatório é legível e contém resumo, análise e status geral.
