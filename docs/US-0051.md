# US-0051 — Classificação por alias do tipo

## Título

TASK-0075: Segunda camada do Classification Engine — alias do tipo (Symbol.Name)
após a categoria Revit (Classification Engine + Knowledge Engine, EPIC-0005).

## Objetivo

Implementar o fallback de classificação por alias do tipo: se a categoria não
resolve, usa `FamilyInstance.Symbol.Name` consultando o Engineering Dictionary.
Sem Family.Name, Parâmetros ou IA.

## Critérios de Aceite

- [x] Tomadas (tipo contendo tomada/tug/outlet/socket) → Outlet.
- [x] Interruptores → Switch.
- [x] Luminárias → LightingFixture.
- [x] Painéis continuam pela categoria.
- [x] Circuitos continuam pela categoria.
- [x] Sem uso de Family.Name / Family.Family.Name.
- [x] Nenhuma Rule alterada.
- [x] Nenhuma IA.

## Fluxo

```
1) Categoria Revit        → Confidence 100
2) Alias do tipo (Symbol.Name, via ClassificationRepository.FindByAlias)
                          → Confidence 80
3) Unknown                → Confidence 0
```

- Reason categoria: "Classificado pela categoria Revit."
- Reason alias: "Classificado pelo alias do tipo."
- Reason unknown: "Categoria ainda não mapeada."

## Detalhes

- `ElementClassifier.Classify(Element)` primeiro resolve por categoria; se Unknown
  e o elemento for `FamilyInstance`, usa `fi.Symbol.Name` para consultar
  `ClassificationRepository.FindByAlias` e, como fallback de "contém", itera
  `GetAllAliases` (case-insensitive). Apenas `Symbol.Name` é usado.
- `aliases.json` já mapeia Outlet/Switch/LightingFixture/Panel/Circuit.
