# US-0052 — Classificação por alias da família

## Título

TASK-0076: Terceira camada do Classification Engine — alias da família
(`Family.Name`) após categoria e alias do tipo (Classification Engine +
Knowledge Engine, EPIC-0005).

## Objetivo

Adicionar o fallback de classificação por alias da família: se categoria e tipo
não resolvem, usa `FamilyInstance.Symbol.Family.Name` consultando o Engineering
Dictionary. Sem Parâmetros nem IA.

## Critérios de Aceite

- [x] Famílias "Tomadas Hospital" → Outlet.
- [x] Famílias "Interruptores" → Switch.
- [x] Famílias "Luminárias" → LightingFixture.
- [x] Painéis continuam pela categoria.
- [x] Circuitos continuam pela categoria.
- [x] Sem uso de parâmetros / LookupParameter / Shared Parameters.
- [x] Sem IA.
- [x] Nenhuma Rule alterada.

## Fluxo

```
1) Categoria Revit                 → Confidence 100
2) Alias do Tipo (Symbol.Name)     → Confidence 80
3) Alias da Família (Family.Name)  → Confidence 70
4) Unknown                         → Confidence 0
```

- Reason categoria: "Classificado pela categoria Revit."
- Reason tipo: "Classificado pelo alias do tipo."
- Reason família: "Classificado pelo alias da família."
- Reason unknown: "Categoria ainda não mapeada."

## Detalhes

- `ElementClassifier.Classify(Element)` aplica, nesta ordem: categoria →
  `Symbol.Name` (`FindByAlias` + contains) → `Symbol.Family.Name`
  (`FindByAlias` + contains). Apenas `Symbol.Name` e `Family.Name` são usados.
- `aliases.json` já contém tomada/interruptor/luminária, então "Tomadas Hospital",
  "Interruptores", "Luminárias" casam por contains.
