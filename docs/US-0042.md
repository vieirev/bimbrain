# US-0042 — Explicar elemento selecionado

## Título

TASK-0066: Primeira funcionalidade real do Copilot — explicar o elemento
selecionado (AI Engine sobre Foundation + Graph, EPIC-0004).

## Objetivo

Implementar o comando `Explicar` (grupo Copilot da Ribbon) que explica o
elemento selecionado no Revit, inteiramente a partir do Project Graph, sem IA,
sem Ollama, sem `QuestionProcessor` e sem acesso direto a `Document` para
relações.

## Critérios de Aceite

- [x] Selecionar qualquer elemento.
- [x] Clicar `Explicar` e receber a explicação.
- [x] Sem IA (sem Ollama).
- [x] Sem `QuestionProcessor`.
- [x] Sem regras.
- [x] 100% Graph (`ProjectGraph`, `ProjectGraphQuery`, `ProjectImpactAnalyzer`, `ElementExplanationService`).

## Fluxo

```
Botão Ribbon (Explicar)
  ↓ ExplainSelectionCommand
  ↓ ProjectGraphBuilder.Build()
  ↓ CopilotContextBuilder.Build(elementId)
  ↓ SelectionContextService.Create(elementId)
  ↓ ElementExplanationService.Explain(node)
  ↓ TaskDialog
```

## Comportamento

1. Sem seleção → `"Selecione um elemento primeiro."`
2. Com seleção → monta `ProjectGraph` e constrói `CopilotContext`.
3. Obtém `SelectionContext` (via `SelectionContextService`).
4. Executa `ElementExplanationService.Explain(node)`.
5. Exibe `TaskDialog` com: Nome, Categoria, Modelo, Nível, Hierarquia,
   Dependências, Elementos afetados e Resumo (`BuildSummary()`).
6. Elemento fora das categorias mapeadas (Projeto, Modelo, Nível, Painel,
   Circuito) → aviso de que não está mapeado no Project Graph.

## Detalhes Técnicos

- `ExplainSelectionCommand` (`IExternalCommand`) substitui `PlaceholderCommand`
  no botão `Explicar` da Ribbon.
- `App.MakeButton(name, text, commandType, iconKey, tooltip, longDescription)`
  gera `PushButtonData` reutilizável para comandos reais (usado pelo botão
  `Explicar`); `MakePlaceholder` passa a delegar a `MakeButton`.
- O comando instancia `ProjectGraphBuilder`, `ProjectGraphQuery`,
  `ProjectImpactAnalyzer`, `ElementExplanationService` e `SelectionContextService`,
  depois `CopilotContextBuilder`, seguindo o pipeline oficial.
- `BuildDialogText` monta o texto do `TaskDialog` com a hierarquia obtida de
  `ElementExplanation.Dependencies` (ancestors) + nó, e o resumo via
  `ElementExplanationService.BuildSummary(node)`.
