# US-0045 — Knowledge Viewer

## Título

TASK-0069: Infraestrutura que conecta uma Rule ao seu documento oficial de
conhecimento (Knowledge Engine, EPIC-0005). Sem IA.

## Objetivo

Criar `KnowledgeDocument` e `KnowledgeRepository` para localizar e carregar a
documentação de uma regra a partir de `RuleName`, e expor essa documentação na
`RuleResultsWindow` via um novo botão `Conhecimento` que abre a `KnowledgeWindow`.

## Critérios de Aceite

- [x] Executar `NBR 5410`.
- [x] Selecionar uma Rule.
- [x] Clicar `Conhecimento`.
- [x] Abrir documentação correspondente.
- [x] Se não existir documentação: exibir `"Documentação não encontrada."`
- [x] Nenhuma Rule alterada.
- [x] `RuleRunner` não alterado.
- [x] Sem IA; Markdown não interpretado (exibido como texto).

## Detalhes Técnicos

- `KnowledgeDocument`: `Title`, `RuleId`, `Standard`, `Markdown`, `Exists`.
- `KnowledgeRepository.GetByRuleName(ruleName)`:
  - deriva o padrão (`NBR5410`) a partir do `ruleName` (antes do `-`);
  - resolve `knowledge/standards/<Standard>/rules/<RuleName>.md`;
  - busca a pasta `knowledge` a partir de `AppDomain.CurrentDomain.BaseDirectory`,
    subindo até 8 níveis (permite encontrar a documentação mesmo quando o addin
    está em `Addins/2025/knowledge`);
  - extrai o `Title` da primeira linha `#` do Markdown;
  - retorna `Exists = false` se o arquivo não for encontrado.
- `KnowledgeWindow` (WPF code-only, `ScrollViewer` + `TextBlock`): exibe o
  `Markdown` como texto puro.
- `RuleResultsWindow` ganha o botão `Conhecimento` (habilitado apenas com regra
  selecionada); `OnKnowledge` chama o repositório e abre `KnowledgeWindow` ou
  mostra a mensagem de ausência.
- Nenhuma Rule alterada; `RuleRunner` inalterado; sem IA.
