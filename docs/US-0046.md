# US-0046 — Rule Inspector

## Título

TASK-0070: Inspector visual dos elementos afetados por uma Rule (Rule Engine,
EPIC-0003). Sem IA, sem alteração nas Rules.

## Objetivo

Criar `RuleInspectorWindow`, que lista os elementos afetados por um `RuleResult`
(Id, Nome, Categoria, Família, Tipo, Nível) e permite selecionar/navegar até o
elemento escolhido individualmente. Integrar o botão `Inspector` na
`RuleResultsWindow`.

## Critérios de Aceite

- [x] Executar uma Rule.
- [x] Selecionar um resultado.
- [x] Abrir o Inspector.
- [x] Listar todos os elementos afetados.
- [x] Selecionar um elemento.
- [x] `Selecionar` e `Navegar` funcionam individualmente (apenas o elemento escolhido).
- [x] Nenhum `TaskDialog`; toda interação via janelas próprias do BIMBrain.

## Detalhes Técnicos

- `RuleInspectorWindow(UIDocument uiDoc, RuleResult result)` percorre
  `result.AffectedElements`; para cada `ElementId` usa `Document.GetElement` para
  obter Id, Nome, Categoria, Família (`FamilyInstance.Symbol.Family`), Tipo
  (`FamilyInstance.Symbol`) e Nível (`Element.LevelId`, fallback
  `FAMILY_LEVEL_PARAM`).
- Lista WPF `ListView`+`GridView` (colunas Id, Nome, Categoria, Família, Tipo,
  Nível); seleção preenche o painel inferior de detalhes.
- `Selecionar` → `uiDoc.Selection.SetElementIds(new List<ElementId>{ item })`.
- `Navegar` → `uiDoc.ShowElements(new List<ElementId>{ item })`.
- `RuleResultsWindow` ganha o botão `Inspector` (habilitado com regra selecionada)
  que abre `RuleInspectorWindow(_uiDoc, _selected)`.
- Não altera `RuleRunner`, `RuleCatalog`, `EngineeringRule`, `QuestionProcessor`
  nem o Copilot; sem IA.
