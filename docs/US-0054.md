# US-0054 — ModelIntegrityAnalyzer consome RuleResult estruturado

TASK-0078: Remover a dependência de parsing textual (Regex) existente no
`ModelIntegrityAnalyzer`. O analyzer passa a consumir dados estruturados
produzidos pelas Rules (`RuleResult.Data`), nunca interpretando `Message`.

## Engine

Rule Engine / Model Engine

## EPIC

EPIC-0003

## User Story

US-0054

## Objetivo

Eliminar completamente o `Regex` do `ModelIntegrityAnalyzer`. As Rules expõem
dados estruturados (`RuleResult.Data`) que são consumidos internamente pelo
analyzer.

## Alterações

- `src/BIMBrain/Rules/RuleResult.cs` — adicionada propriedade
  `Dictionary<string, object> Data` (bolsa estruturada genérica).
- `src/BIMBrain/Rules/Consistency/ElementsWithoutCircuitRule.cs` — cada
  `RuleResult` de categoria sem circuito passa a preencher `Data` com
  `Tipo` (tomadas/luminárias/interruptores) e `Count` (quantidade de
  elementos não conectados).
- `src/BIMBrain/Rules/Standards/NBR5410/NBR5410_001_ConnectedOutletsRule.cs`
  — filtro de tomadas passa a usar `Data["Tipo"] == "tomadas"` em vez de
  `Message.Contains("tomada")` (comportamento idêntico).
- `src/BIMBrain/ModelIntegrityAnalyzer.cs` — remove `Regex` e `using
  System.Text.RegularExpressions`; lê `Data["Tipo"]`/`Data["Count"]` para
  compor `ModelIntegrityResult`.

## Resumo técnico

- `RuleResult.Data` é a fonte canônica de dados estruturados entre Rules e
  consumidores internos (inclusive regras normativas que filtram resultados).
- `ModelIntegrityAnalyzer.Analyze()` extrai contagens por categoria via
  `Data["Tipo"]`/`Data["Count"]`, mantendo o contrato público
  `ModelIntegrityResult` inalterado.
- O `Regex` (`\d+` sobre `Message`) foi eliminado; o analyzer não depende mais
  de formatação de texto.
